terraform-gcp-cloud-run Architecture Document

Cloud Run Module

Executive Summary

A Terraform module for deploying serverless container workloads on Google Cloud Run v2. Manages services with auto-scaling (0 to 100 instances), jobs with parallelism and retry policies, VPC Access Connectors for private network connectivity, Secret Manager integration for secure environment variables, IAM bindings for authenticated/unauthenticated access, and custom domain mappings. Runs on 2nd-gen execution environment with configurable CPU, memory, concurrency, and timeout settings.

Component Breakdown

google_cloud_run_v2_service.this

Cloud Run v2 services with configurable container image, port (default 8080), CPU/memory limits, auto-scaling (min 0, max 100), 80 concurrent requests per instance, Gen2 execution environment, environment variables (plain and secret-backed), VPC connector for private egress, and ingress controls (ALL, INTERNAL_ONLY, INTERNAL_AND_CLOUD_LOAD_BALANCING).

google_cloud_run_v2_job.this

Cloud Run v2 jobs for batch workloads with task count, parallelism, max retries (default 3), configurable timeout, VPC connector support, and environment variables.

google_vpc_access_connector.this

Serverless VPC Access connectors with configurable machine type (default e2-micro), instance scaling (2-3), throughput limits (200-300 Mbps), and network/CIDR configuration for private connectivity.

google_cloud_run_service_iam_member.allow_unauthenticated

Conditionally grants roles/run.invoker to allUsers when allow_unauthenticated = true on a per-service basis.

Data Flow

  • HTTP Requests: External traffic arrives via HTTPS, optionally through custom domain mappings, and is routed to Cloud Run services.
  • Service Processing: Containers handle requests with auto-scaling from 0 (scale-to-zero) to 100 instances based on demand.
  • Private Access: Services connect to VPC resources (Cloud SQL, Memorystore, internal APIs) through VPC Access Connectors with PRIVATE_RANGES_ONLY or ALL_TRAFFIC egress.
  • Secrets: Sensitive configuration is injected from Secret Manager as environment variables at container startup.
  • Jobs: Batch workloads execute with parallelism, retry on failure, and terminate after timeout.

Industry Adaptation

  • SaaS / Microservices: Multiple services with independent scaling, VPC connectors to shared databases, Secret Manager for API keys.
  • Data Processing: Cloud Run jobs for ETL batch workloads with retry policies and parallelism.
  • API Backends: Authenticated APIs with custom domains, auto-scaling to handle traffic spikes, VPC connectivity for database access.
  • Event-Driven: Scale-to-zero services triggered by Pub/Sub, Eventarc, or Cloud Scheduler for cost-efficient event processing.

Production Readiness Checklist

  • Set min_instance_count >= 1 for latency-sensitive services to avoid cold starts
  • Assign dedicated service accounts with least-privilege IAM roles
  • Use Secret Manager for all sensitive environment variables
  • Configure VPC Access Connector for private database and API connectivity
  • Set ingress = "INGRESS_TRAFFIC_INTERNAL_AND_CLOUD_LOAD_BALANCING" for internal services
  • Disable allow_unauthenticated unless the service is a public API
  • Set appropriate CPU/memory limits based on load testing
  • Configure meaningful labels for cost tracking and resource organization
  • Set max_instance_count to prevent runaway scaling costs
  • Configure appropriate timeouts for service and job workloads

Deployment

module "cloud_run" {
  source = "github.com/kogunlowo123/terraform-gcp-cloud-run"

  project_id = "my-project"
  region     = "us-central1"

  vpc_connectors = {
    "main-connector" = {
      network       = "projects/my-project/global/networks/default"
      ip_cidr_range = "10.8.0.0/28"
    }
  }

  services = {
    "my-api" = {
      image              = "us-central1-docker.pkg.dev/my-project/repo/api:v1"
      cpu                = "2000m"
      memory             = "1Gi"
      min_instance_count = 1
      max_instance_count = 10
      service_account    = "api-sa@my-project.iam.gserviceaccount.com"
      vpc_connector      = "main-connector"
      env_vars           = { LOG_LEVEL = "info" }
      secret_env_vars    = {
        DB_PASSWORD = { secret = "db-password", version = "latest" }
      }
      allow_unauthenticated = false
    }
  }

  jobs = {
    "nightly-etl" = {
      image       = "us-central1-docker.pkg.dev/my-project/repo/etl:v1"
      task_count  = 5
      max_retries = 2
      timeout     = "1800s"
    }
  }

  labels = { team = "platform", env = "production" }
}
